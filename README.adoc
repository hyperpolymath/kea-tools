== Overview

Kea-Tools is the unified monorepo consolidating the four core components of the Kea ecosystem into a single development and deployment surface.

The *Kea ecosystem* provides intelligent infrastructure orchestration, investigation, signalling, and WebAssembly interface tooling. Each component corresponds to a behavioural trait of the Kea parrot: strategic roosting, vocal coordination, investigative dexterity, and adaptive intelligence.

== Components

[cols="1,2,3"]
|===
|Directory |Component |Purpose

|`bivouac/`
|**Kea-Bivouac** -- The Command Authority
|Orchestration and deployment controller. Manages the separation of administration from runtime, executes playbooks, enforces mTLS zero-trust communication.

|`call/`
|**Kea-Call** -- The Contact-Signalling Protocol
|Cap'n Proto and MCP definitions for ecosystem communication. Zero-copy binary serialisation between sensors (Mandible) and logic (Wit) via the Request-Signal-Action loop.

|`mandible/`
|**Kea-Mandible** -- The Investigative Mandible
|High-dexterity sensor suite (Kea-Beak, WP-Praxis, Slop-Gate). Probes filesystems, network configurations, and WordPress cores to identify instability and bloat.

|`wit/`
|**Kea-Wit** -- WebAssembly Interface Types
|WIT interface definitions and tooling for WebAssembly Component Model development, validation, and code generation.
|===

== Quick Start

[source,bash]
----
# Build all Rust components
just build

# Run all tests
just test

# Deep audit a target path (via Mandible)
just mandible-pry /var/www/html

# Execute a failover playbook (via Bivouac)
just bivouac-playbook integrity-violation
----
